1.컨트롤러에 인터페이스 적용
각 컨트롤러는 main() 함수에서 시작하기 때문에 interface를 만들어서
main()을 만들게한다.
interface는 상속받을시 implements 키워드를 사용한다.

2.추상클래스
interface와 별개로 일반 메소드가 필요할 경우에는 추상클래스를 사용한다.
클래스에 abstract 키워드가 붙으면 그 클래스는 추상클래스가 된다.
메소드에 abstract 키워드가 붙으면 그 메소드는 추상메소드(=만들어야하는메소드)가 된다.
일반 메소드를 만들어 놓으면 그 메소드는 상속받아서 사용할 수 있다.
추상클래스를 상속시 extends 키워드를 사용한다.

3.Goods.php
상품페이지 역할을 하는 Goods.php와 뷰로 뿌려줄 Goods.html 을 만들고
table에 goods를 추가해준다. 컬럼 ( id , images , price) 설정
마지막으로 DB 호출

4.bootstrap 적용
부트스트랩을 적용시켜 상품목록을 꾸며준다.

5.상품 목록 삽입 기능 추가 1
Goods_new.html을 만들고 Goods.php에 Uri를 검사해서 상품 삽입과 상품 목록 출력을 하는 분기점을 만든다.

6.상품 목록 삽입 기능 추가 2
form에서 파일을 업로드 할 경우 html 에서 form에 enctype="multipart/form-data" 를 작성해야한다.
php에서는 move_uploaded_file(임시저장되어있는경로 , 실제경로와파일이름) 을 사용해야한다.

7.상품 상세 페이지 추가
상품명 혹은 상품의 이미지를 누르면 상품의 상세설명 페이지로 이동하도록 만들어준다.

8.상품 조회수 추가 및 순서 정렬
상세설명 페이지로 이동하면 조회수가 증가하도록 한다.
조회수를 테이블에 컬럼으로 추가해주고 페이지에 접속할때마다 조회수가 증가하도록 한다.
상품 목록에서 조회수로 오름차 정렬을 하면 인기순으로 정렬이된다.

9.카테고리 테이블 생성 및 수정
카테고리 분류를 위해서 카테고리 테이블을 만들고 카테고리 함수를 만든다.

10.장바구니 추가
상품 상세 페이지에서 담기 버튼을 누르면 장바구니 테이블에 추가되는 기능을 추가한다.

11.로그인 + 장바구니
로그인을 한 상태로 장바구니에 담으면 테이블에 email에 들어간다.

12.장바구니에 들어간
로그인 상태에서 담기를 누르면 어떤 계정에 물건이 담겼고
어떤 물건이 담겼는지에 대한 쿼리를 출력한다.
