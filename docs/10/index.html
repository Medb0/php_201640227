10주차 리포트

1.index.php
이전 프로젝트 정리가 잘 안된관계로 프로젝트를 다시 만들어서 정리하는중이다.

2.Loading.php
spl_autoload_register 함수를 사용해서 필요한 클래스를
자동으로 require 해주는 함수를 만들어 논다.

3.Table.php
이전 프로젝트의 Table을 생성하는 클래스를 가져옴.

4.index.php 수정 + Database.php 추가
Database.php에서 DB의 Table 값을 불러오고 index.php의 table.html을 불러와서 수정한다.

5.bootstrap jumbotron 적용
jumbotron을 적용시켜 list 모양을 깔끔하게 바꿈.

6.index , table 수정
table에 있던 html 코드를 index에서 한꺼번에 처리하게끔 수정한다.

7.HtmlTable.php
index.php는 컨트롤러의 역할이다.
컨트롤러의 역할 중에 html이 섞이면 MVC패턴이 흐지부지되기 떄문에
HtmlTable.php를 따로 만들어서 뷰에 뿌려주는 컨트롤러를 따로 만들어준다.

8.URI 제어
URI 폴더에 들어가는 방식이 아닌 어디서 들어가든지 그 폴더가 최상위가 되도록 사전작업 및 테스트를 하는 코드이다.
URI 주소를 "/" 로 구분하여 배열에 넣는다.

9.URI 제어 2
URI를 나누어 배열에 넣어두고 그 배열값을 비교하여 특정 URI가 나올시 그에 맞는 컨트롤러를 작동시키거나
조건에 맞지 않을시 처음페이지로 보낸다.

10.URI 제어 3
URI에 맞춰 실제로 Table 컨트롤러를 실행한다.

11. TableName 클릭시 반응 추가
Table에서 출력된 값을 클릭할 경우 그 Table의 콜롬값을 출력하는 페이지로 이동시키는 a 태그를 추가한다.

12.desc.html 추가
테이블명 클릭시 이동되는 컨트롤러를 만들었으니 세부내용이 출력될 뷰를 만들어준다.

13.부트스트랩 적용 및 body 생성 코드 최적화
desc.html에 부트스트랩을 적용한다.
body 부분에 출력된 값을 테이블로 만들어서 정렬되게끔 코드로 작성한다.
